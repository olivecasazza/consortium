Like run_cascade but also emits fine-grained CascadeEvents through events. Use this for live tracing, JSONL persistence, or any consumer that wants per-edge timing rather than per-round aggregates.

trace and events are independent — passing both is fine, useful for migrating from the snapshot model to the event model incrementally.
