A Task manages a set of workers and their results.

The Task is the primary orchestrator in consortium. It schedules workers for execution, runs them (respecting fanout limits), and collects their output, return codes, and timeout information.

§Design Differences from Python

The Python Task is heavily thread-based with a thread-local singleton pattern (task_self()). The Rust version is single-threaded and owned, using explicit lifetimes rather than thread-local storage. Multi-threaded usage should wrap Task in Arc<Mutex<Task>> if needed.
